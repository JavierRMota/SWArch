Domain-Specific Language Pattern¶ ↑

This documentation was developed as part of a programming activity for the Software Design and Architecture course (TC3049), at the Tecnológico de Monterrey, Campus Estado de México.

Overview¶ ↑

The Domain-Specific Language Pattern suggests that you should focus on the language itself, not on the interpreter. It says that sometimes you can make problems easier to solve by giving the user a convenient syntax for expressing the problem in the first place.
